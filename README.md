# domain-driven-design-skill

Domain-Driven Design skill for strategic DDD, tactical modeling, bounded
contexts, context maps, aggregates, domain events, ubiquitous language, and
event storming.

## Install For Codex

```powershell
npx -y skills add MartinEgli/domain-driven-design-skill --skill * -a codex --yes
```

## Checks

```powershell
npm run validate
npm run test
npm run package
```

## Continuous Improvement

This skill supports an explicit feedback loop. Use /domain-driven-design feedback to
capture lessons and /domain-driven-design improve to propose changes. Improvements must
remain traceable, validated, committed on a feature branch, and pushed before
they become part of the released skill behavior.
